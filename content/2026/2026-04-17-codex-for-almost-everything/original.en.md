# Codex for (almost) everything

Source: https://openai.com/index/codex-for-almost-everything/
Saved: 2026-04-17

OpenAI says it is releasing a major update to Codex, describing it as a more capable partner for the more than 3 million developers who use it weekly across the software development lifecycle.

According to the announcement, Codex can now operate a computer alongside the user, work with more apps and tools, generate images, remember preferences, learn from prior actions, and take on ongoing or repeatable work. The Codex app is also described as adding deeper support for developer workflows such as PR review, viewing multiple files and terminals, connecting to remote devboxes via SSH, and using an in-app browser to iterate on frontend work, apps, and games.

## Computer use and browser use

The post says Codex can now use all apps on the user’s computer through background computer use, seeing, clicking, and typing with its own cursor. OpenAI says multiple agents can work on a Mac in parallel without interfering with the user’s own work.

The announcement also says Codex is beginning to work natively with the web. The app now includes an in-app browser where users can comment directly on pages to guide the agent more precisely. OpenAI says this is especially useful today for frontend and game development, with broader browser control planned over time.

## Images, plugins, and richer workflows

The post says Codex can now use `gpt-image-1.5` to generate and iterate on images. Combined with screenshots and code, this is positioned as useful for product concepts, frontend designs, mockups, and games.

OpenAI also says it is releasing more than 90 additional plugins that combine skills, app integrations, and MCP servers. The announcement specifically names Atlassian Rovo, CircleCI, CodeRabbit, GitLab Issues, Microsoft Suite, Neon by Databricks, Remotion, Render, and Superpowers.

The app is also described as adding support for addressing GitHub review comments, running multiple terminal tabs, and connecting to remote devboxes over SSH in alpha. It can also open files in a sidebar with rich previews for PDFs, spreadsheets, slides, and docs, while a summary pane tracks plans, sources, and artifacts.

## Automations, memory, and proactive suggestions

OpenAI says automations have been expanded to allow reusing existing conversation threads so context can be preserved over time. Codex can also schedule future work for itself and wake up automatically to continue long-running tasks across days or weeks.

The post says teams are already using automations for tasks such as landing open pull requests, following up on tasks, and staying on top of conversations in tools like Slack, Gmail, and Notion.

It also announces a preview of memory, which is described as allowing Codex to remember useful context from prior experience, including user preferences, corrections, and hard-won information. OpenAI says this should make future tasks faster and higher quality.

Codex also now proactively suggests useful follow-up work. OpenAI describes a future where Codex can use project context, connected plugins, and memory to suggest how to begin the work day or resume a prior project, including identifying open comments in Google Docs, pulling relevant context from Slack, Notion, and code, and presenting a prioritized action list.

## Rollout and positioning

OpenAI says these updates are rolling out starting now to Codex desktop app users signed in with ChatGPT. It adds that personalization features such as memory and context-aware suggestions will roll out later to Enterprise, Edu, and users in the EU and UK, while computer use is initially available on macOS and will also expand to the EU and UK later.

The post frames this release as part of a broader expansion in how developers use Codex, from code generation into understanding systems, gathering context, reviewing work, debugging issues, coordinating with teammates, and keeping long-running work moving.

# Summary

Cloudflare’s post introduces an experimental voice pipeline for the Agents SDK so developers can add real-time STT and TTS to the same agent architecture they already use for text, state, tools, and persistence. The main idea is that voice should be another interface to the same agent, not a separate framework.

The article also positions voice as part of a broader agent runtime, highlighting shared state between text and voice, streaming responses, scheduling, tools, Twilio integration, and interchangeable provider layers.

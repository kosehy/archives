# 요약 — S3 Files와 S3의 변화하는 모습

## 핵심 아이디어

이 글은 현대 워크로드에서 스토리지의 핵심 문제를 용량이나 내구성보다, 서로 다른 데이터 접근 방식 사이의 마찰에서 찾는다. S3 Files는 기존 S3 데이터를 파일시스템 인터페이스로 직접 다룰 수 있게 해 그 마찰을 줄이려는 시도로 제시된다.

## 핵심 포인트

- 글은 유전체 분석 사례를 통해, 클라우드 병렬 계산에는 S3가 맞지만 실제 도구는 로컬 파일시스템을 전제로 만들어졌다고 설명한다.
- 그 결과 데이터 복사, 중복 저장, 운영 오버헤드가 반복되며 저자는 이를 데이터 마찰이라고 부른다.
- 에이전트 기반 개발은 애플리케이션 생성 속도를 높이기 때문에, 쉽게 붙일 수 있는 데이터 계층의 중요성을 더 키운다고 본다.
- S3 Tables와 S3 Vectors는 S3가 더 높은 수준의 데이터 프리미티브로 확장된 선행 사례로 제시된다.
- S3 Files는 EFS와 S3를 통합해 객체를 파일처럼 마운트하고 사용할 수 있게 한다.
- 동시에 파일과 객체 의미론은 수정, 공유, 이벤트 처리 방식이 달라 통합 설계가 기술적으로 매우 어렵다는 점도 강조한다.

## 한 줄 결론

이 글은 S3 Files를 통해 S3가 다양한 데이터 접근 패턴을 흡수하는 더 넓은 데이터 플랫폼으로 진화하고 있다고 설명한다.

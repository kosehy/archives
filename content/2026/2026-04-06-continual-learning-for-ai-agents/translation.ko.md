# AI 에이전트를 위한 지속적 학습

원문: https://blog.langchain.com/continual-learning-for-ai-agents/
아카이브일: 2026-04-06
언어: 한국어
문서 유형: 에이전트 시스템 / 지속적 학습 / LangChain 에세이

## 핵심 주장

이 글은 agentic 시스템에서의 지속적 학습을 단순히 모델 가중치 업데이트로만 보면 안 된다고 주장한다.

대신 에이전트는 시간이 지나며 다음 세 층위에서 개선될 수 있다고 본다.

- 모델
- 하네스
- 컨텍스트

핵심은 이 세 층위를 분리해서 봐야, AI 시스템의 개선과 기억, 운영 학습을 더 정확하게 설계할 수 있다는 점이다.

## 세 가지 층위

### 1. 모델 레이어

모델 레이어는 모델 가중치 자체를 뜻한다.

대부분의 사람이 지속적 학습이라고 하면 가장 먼저 떠올리는 층위가 이것이다. 글은 SFT, 강화학습 같은 적응 기법을 언급한다.

여기서 핵심 문제는 catastrophic forgetting이다. 새 데이터나 새 작업으로 모델을 업데이트할수록, 기존에 알고 있던 능력이 약해질 수 있다는 뜻이다.

또 글은 agentic 시스템을 위해 모델을 따로 학습시키는 경우에도, 보통은 개별 사용자보다 에이전트 전체 수준에서 이루어진다고 본다.

### 2. 하네스 레이어

하네스 레이어는 에이전트를 움직이는 코드와, 항상 켜져 있는 기본 지시문·도구를 뜻한다.

글은 최근 하네스를 직접 최적화하려는 흐름을 소개한다. 예를 들어 Meta-Harness처럼, 에이전트를 여러 과제에 돌리고 로그를 평가한 뒤, 그 트레이스를 다시 코딩 에이전트가 분석해서 하네스 코드를 바꾸도록 하는 방식이다.

핵심 아이디어는 에이전트가 더 나아지는 방식이 모델 변경만이 아니라, 모델이 쓰이는 실행 루프와 주변 시스템을 바꾸는 것일 수도 있다는 점이다.

### 3. 컨텍스트 레이어

컨텍스트는 하네스 바깥에 있으면서, 하네스의 동작을 조정하는 설정성 재료를 뜻한다.

예를 들면 다음이 포함된다.

- 지시문
- 스킬
- 도구
- 메모리

글은 컨텍스트 학습이 여러 범위에서 일어날 수 있다고 본다.

- 에이전트 수준
- 사용자 또는 테넌트 수준
- 조직 또는 팀 수준

또 이런 업데이트는 두 방식으로 일어날 수 있다고 설명한다.

- 최근 트레이스를 분석해 사후적으로 컨텍스트를 업데이트하는 오프라인 방식
- 에이전트가 실제로 일하는 중에 핫패스에서 메모리를 업데이트하는 온라인 방식

이 글은 OpenClaw를, 시간이 지나며 `SOUL.md`를 업데이트하는 에이전트의 예시로 직접 언급한다.

## 예시 매핑

글은 두 가지 구체적 매핑도 제시한다.

Claude Code의 경우:
- model = claude-sonnet 등 모델
- harness = Claude Code 자체
- user context = `CLAUDE.md`, `/skills`, `mcp.json`

OpenClaw의 경우:
- model = 다양한 모델
- harness = Pi와 그 주변 스캐폴딩
- agent context = `SOUL.md`, ClawHub의 skills

이 매핑이 유용한 이유는, 사용자가 느끼는 ‘에이전트의 지능’이 전부 베이스 모델에 속하는 것이 아니라는 점을 분명하게 보여주기 때문이다.

## 트레이스가 공통 기반이라는 점

이 글의 가장 좋은 포인트 중 하나는 트레이스를 세 가지 지속적 학습 흐름의 공통 기반으로 본다는 점이다.

트레이스는 에이전트가 실제로 무엇을 했는지의 전체 실행 경로를 기록한다. 이 기록은 다음과 같은 용도로 다시 쓸 수 있다.

- 모델 학습 또는 적응
- 하네스 개선
- 메모리나 컨텍스트의 시간 경과 업데이트

글은 LangSmith를 LangChain이 이런 트레이스를 수집하고 활용하는 플랫폼으로 제시한다.

## 왜 이 프레이밍이 중요한가

이 글의 가치는 주로 개념 정리에 있다.

보통 ‘학습’이라는 말로 한데 뭉뚱그려지는 세 가지 다른 개선 경로를 분리해 준다. 이 구분은 중요하다. 각 레이어는 서로 다른 tradeoff를 갖기 때문이다.

- 모델 업데이트는 비용이 크고 forgetting 위험이 있다.
- 하네스 업데이트는 공통 실행 논리를 바꾼다.
- 컨텍스트 업데이트는 더 유연하고, 국소적이며, 운영적으로 실용적일 때가 많다.

이 구분은 무엇을 어디서 얼마나 자주 바꿔야 하는지를 훨씬 더 현실적으로 생각하게 해 준다.

## 최종 takeaway

이 글의 핵심 메시지는 에이전트를 위한 지속적 학습이 하나의 단일한 것이 아니라는 점이다.

에이전트는 가중치를 바꾸면서도, 하네스를 바꾸면서도, 그리고 시간이 지나며 행동을 형성하는 컨텍스트 메모리와 설정을 바꾸면서도 개선될 수 있다.

이 세 층위를 분리해 인식하는 것이 더 나은 시스템 설계와 학습 전략으로 이어진다.

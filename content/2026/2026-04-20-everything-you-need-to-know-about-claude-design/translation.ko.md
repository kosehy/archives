# Claude Design에 대해 알아야 할 모든 것

원문: https://getpushtoprod.substack.com/p/everything-you-need-to-know-about
저장일: 2026-04-20
번역 메모: 현재 접근 가능한 본문을 기준으로 자연스럽게 옮긴 번역본.

이 Substack 글은 Anthropic의 Claude Design 프리뷰를 몇 시간 동안 체계적으로 직접 써본 뒤 정리한 실사용 워크스루다. 단순히 화제에 반응하는 글이 아니라, 실제 기능을 하나씩 테스트해 본 사람의 시점에서 쓰였다.

글의 전체 프레이밍은 Claude Design을 AI 보조 디자인 도구로서 꽤 강력한 신규 진입자로 보되, 아직은 전문 디자이너용 Figma나 Adobe 대체재는 아니라고 본다. 대신 지금 당장 빠른 프로토타입, 슬라이드, 애니메이션, 그리고 “충분히 괜찮은” 시각 결과물이 필요한 엔지니어, 크리에이터, 기술 실무자에게는 이미 상당히 유용하다는 것이 핵심 주장이다.

## 핵심 인터페이스 변화

첫 번째 큰 관찰은 레이아웃과 상호작용 모델이다. Claude Design에서는 채팅이 왼쪽에 있고, 오른쪽의 캔버스가 화면 대부분을 차지한다. 글쓴이는 이 구조가 사용자의 사고방식을 바꾼다고 본다. 단순히 AI에 프롬프트를 던지는 느낌보다, 실제 디자인을 조정하고 있고 채팅은 그 조정을 위한 제어면처럼 느껴진다는 것이다.

이 변화는 미묘하지만 중요하다고 설명된다. 디자인 자체가 제품의 중심이 되고, 대화는 그것을 반복 수정하기 위한 인터페이스가 된다.

## 프로젝트 유형과 시작 모드

글에 따르면 Claude Design은 현재 새 프로젝트를 시작할 때 네 가지 템플릿을 제공한다.

- Prototype
- Slide Deck
- From Template
- Other

Prototype 안에서는 Wireframe과 High Fidelity를 고를 수 있다. 전자는 빠른 스케치와 탐색용이고, 후자는 디자인 시스템과 브랜드 자산을 활용한 더 완성도 높은 시안용이라고 설명한다.

Slide Deck 모드에는 speaker notes 옵션이 있는데, 글쓴이는 이를 특히 높게 평가한다. 슬라이드 자체는 깔끔하게 유지하면서, 세부 설명은 노트로 밀어 넣을 수 있기 때문이다.

From Template는 내장 애니메이션 템플릿을 포함하고 있으며, 기존 프로젝트를 재사용 가능한 템플릿으로 바꾸는 것도 가능하다고 설명한다. Other는 실험용 빈 캔버스에 가깝다.

## 앞단의 맥락 수집

이 글이 흥미롭게 보는 부분 중 하나는 Claude Design이 단순한 blank canvas에서 시작하지 않는다는 점이다. 생성 전에 스타일, 색 방향, 레이아웃 밀도 같은 몇 가지 질문을 하는 짧은 wizard를 먼저 실행한다. 글쓴이는 이것을 coding 도구에서의 context engineering과 비슷하게 해석한다. 앞단에서 맥락을 더 많이 모을수록 첫 결과가 더 잘 나오고 이후 수정 횟수가 줄어든다는 것이다.

## 예시 프로젝트와 생성 범위

Anthropic은 제품 안에 예시 프로젝트도 함께 넣어두었다고 한다. 글은 interactive shader wallpaper, animation 예시, globe loader, text particle effect 등을 언급하며, 이 예시들이 도구가 만들 수 있는 범위를 보여준다고 본다. 특히 결과물이 웹 기술 기반으로 브라우저에서 돌아간다는 점도 강조된다.

글쓴이는 유튜브용 애니메이션 워크플로에 특히 관심을 가졌고, 가져온 디자인 시스템 미학이 생성된 모션 작업에도 적용되는 점을 흥미롭게 봤다.

## 편집과 피드백 도구

글은 생성 이후 수정 워크플로도 자세히 본다. 소개되는 주요 캔버스 도구는 다음과 같다.

- **Tweaks**: 프로토타입 내부에 자동으로 생성되는 슬라이더와 색상 컨트롤
- **Edit mode**: 배경색, 폰트, 요소 속성을 바꾸는 속성 편집기
- **Comments**: 특정 요소에 고정된 코멘트를 선택적으로 묶어 Claude에 한 번에 보낼 수 있는 기능
- **Draw mode**: 캔버스 위 직접 주석, 큐잉, 음성 입력까지 가능한 마킹 도구

글쓴이는 특히 comments의 배치 피드백 모델을 높게 평가한다. 실제 팀 작업은 하나씩 고치기보다 여러 피드백을 묶어 전달하는 경우가 많기 때문이다.

다만 동시에, 전문 디자이너라면 Figma 수준의 정밀 제어와 깊이를 그리워할 것이라고도 말한다.

## import와 외부 맥락

글쓴이 기준에서 이 제품의 큰 강점 중 하나는 import 시스템이다. Claude Design은 다음과 같은 외부 맥락을 불러올 수 있다고 한다.

- Figma `.fig` 파일
- GitHub 저장소
- URL의 live web element
- local code folder
- 다른 Claude Design 프로젝트
- 이미지, 문서, 폴더 drag and drop

특히 Figma import와 GitHub 연결이 중요하다고 본다. 이 지점에서 Claude Design이 단순 장난감이 아니라 실무 주변부에서 실제 쓸모 있는 도구로 바뀐다고 설명한다. 새 디자인이 실제 비주얼 시스템과 컴포넌트 패턴을 상속받을 수 있기 때문이다.

## 조직 단위 디자인 시스템

글은 디자인 시스템 기능을 제품의 가장 중요한 요소로 본다. 조직 설정에서 코드베이스, 슬라이드, PDF, 로고, 색상 팔레트, 타이포 자료 등을 올리면 Claude가 반복되는 색, 타이포, 컴포넌트, 레이아웃 패턴을 추출한다고 한다.

디자인 시스템에는 published/draft 구분이 있고, 하나를 기본값으로 지정하면 새 프로젝트가 자동으로 그것을 상속받는다. 글쓴이는 이것이 매 프로젝트를 처음부터 다시 시작하지 않게 해주는 핵심이라고 본다.

코드베이스 연결 역시 중요하게 본다. Claude가 컴포넌트 라이브러리 패턴을 참조할 수 있기 때문에, 디자인과 엔지니어링 사이를 잇는 다리 역할을 한다는 것이다.

## 모델 선택과 export 흐름

글쓴이는 Claude Design이 여러 Claude 모델을 노출하고, 대화 맥락을 잃지 않은 채 모델을 중간에 바꿀 수 있다고 말한다. Opus 4.7은 기본이자 최고 품질 옵션으로, Haiku는 더 빠르지만 약한 옵션으로 설명된다.

export는 ZIP, PDF, PPTX, standalone HTML, Canva handoff 등을 지원한다고 한다. 그중 글쓴이가 가장 좋아한 것은 “Handoff to Claude Code”다. 시각 프로토타입을 Claude Code 세션으로 넘겨 바로 구현 단계로 이어질 수 있기 때문이다.

이 design-to-code 연결이야말로 제품에서 가장 전략적으로 흥미로운 부분으로 설명된다. 장기적으로 중요한 것은 단순 AI 디자인 보조가 아니라, 프로토타이핑과 구현이 분리된 세계가 아니게 되는 파이프라인이라는 해석이다.

## 최종 평가

글쓴이의 결론은 Claude Design이 아직 고정밀 전문 디자인 작업에서 Figma나 Adobe를 대체하진 못한다는 것이다. 디자이너들이 의존하는 정밀 편집과 협업 기능이 많이 부족하다는 판단이다.

하지만 엔지니어, 크리에이터, 프레젠테이션 제작, 모션 실험, 빠른 프로토타입 작업에는 이미 충분히 유용하다고 본다. 특히 design system ingestion과 Claude Code handoff의 조합이 중요하다고 본다. 그런 관점에서 Claude Design은 단독 디자인 도구라기보다 더 큰 design-to-build 파이프라인의 앞단으로 더 의미가 있다는 평가다.
